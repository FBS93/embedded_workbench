## Glossary

| Term | Definition |
|---|---|
| Mechanical requirement | Traceable element of the mechanical requirements defining a requirement to be implemented. |
| Mechanical component | Traceable element of the mechanical architecture representing an indivisible architectural element that addresses one or more needs specified by mechanical requirements. |
| Mechanical interface | Traceable element of the mechanical architecture defining the interaction between mechanical components. |
| Mechanical parameter | Traceable element of the mechanical architecture defining values that affect the mechanical solution. |
| Mechanical integration test | Traceable element validating the mechanical architecture. |
| Mechanical qualification test | Traceable element validating the mechanical requirements. |

## Work products organization

The mechanical domain shall be organized by process. Where applicable, processes shall have dedicated folders containing their generated work products. The structure shall be as follows:

```text
mech/
  req/
  arch/
  src/
    i_test/
      <integration_test_name>/
        doc/
        src/
    q_test/
      <qualification_test_name>/
        doc/
        src/
    <mechanical-component>/
      src/
      export/
    ...
```

The `mech/req/` folder contains the work products generated by the `mech-requirements` process (only Markdown documents: `.md`).
The `mech/arch/` folder contains the work products generated by the `mech-architecture` process (only Markdown documents: `.md`).

The `mech/src/i_test/<integration_test_name>/doc/` folder contains the work products for a specific integration test generated by the `mech-integration-test` process (only Markdown documents: `.md`).
The `mech/src/i_test/<integration_test_name>/src/` folder contains the work products for a specific integration test generated by the `mech-integration-test` process (only Python source files: `.py`).

The `mech/src/q_test/<qualification_test_name>/doc/` folder contains the work products for a specific qualification test generated by the `mech-qualification-test` process (only Markdown documents: `.md`).
The `mech/src/q_test/<qualification_test_name>/src/` folder contains the work products for a specific qualification test generated by the `mech-qualification-test` process (only Python source files: `.py`).

The `mech/src/<mechanical-component>/src/` folder contains the work products for a specific mechanical component generated by the `mech-implementation` process (only Python source files: `.py`).
The `mech/src/<mechanical-component>/export/` folder contains the exported elements for a specific mechanical component generated by the `mech-implementation` process.

Mechanical component folders may be nested to support logical grouping and hierarchical organization according to mechanical complexity. In folders used only for grouping (i.e., not representing a mechanical component), the `src/` and `export/` folders shall not be present.
