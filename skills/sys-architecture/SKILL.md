## When to Use

- After system requirements are approved.
- When any system architecture activity is required.

## Critical Patterns

- Shall follow the provided resources.
- Shall align with system requirements (no inconsistencies).
- Shall ensure all system architecture is derived from system requirements by adding explicit traceability between them.
- Shall prioritize reuse over creation.
- Shall keep documentation DRY (Don't Repeat Yourself) by adding references instead of duplicating content.
- Shall ask for approval on key system architectural decisions.
- Shall request final validation before closing the system architecture to ensure no further iterations are needed.
