# System domain overview

## Glossary

| Term | Definition |
|------|------------|
| System requirement | Traceable element of the system requirements defining a requirement to be implemented. |
| System component | Traceable element of the system architecture representing an indivisible architectural element that addresses one or more needs specified by system requirements. |
| System interface | Traceable element of the system architecture defining the interaction between system components. |
| System parameter | Traceable element of the system architecture defining configurable or fixed values that affect system behavior. |
| System design | Traceable element of the system architecture defining architectural decisions. |
| System integration test | Traceable element validating interactions between system components and/or the defined system architecture. |
| System qualification test | Traceable element validating the system requirements. |

## Work products organization

The system domain shall be organized by process. Where applicable, processes shall have dedicated folders containing their generated work products. The structure shall be as follows:

```text
sys/
  req/
  arch/
  i_test/
  q_test/
```

The `sys/req/` folder contains the work products generated by the `sys-requirements` process.  
The `sys/arch/` folder contains the work products generated by the `sys-architecture` process.
The `sys/i_test/` folder contains the work products generated by the `sys-integration-test` process.
The `sys/q_test/` folder contains the work products generated by the `sys-qualification-test` process.
